# 2.3 資料視覺化：選對圖表

> [!NOTE] 圖解簡報設計 · 單元 2.3

> [!KEY] 一句話總結
> 圖表的任務是**讓結論一秒被看懂**，所以要先想「我要觀眾看出什麼」，再決定用哪種圖。

> [!KEY] 你會學到
> - 四種常見目的各該配哪種圖表
> - 讓圖表更有力的三個技巧
> - 為什麼標題要寫「結論」而不是「圖名」
> - 一張好圖表該長什麼樣

## 內容筆記

### 先想結論，再選圖表
選圖表的順序常常是反的：很多人先畫圖，再想要說什麼。正確的做法是先確定「我要觀眾看出什麼」，再挑最能凸顯那個結論的圖型。不同目的，適合的圖不一樣：

| 你想表達 | 適合的圖 | 提醒 |
|---|---|---|
| 比較大小 | 長條圖 | 由高到低排序更好讀 |
| 隨時間變化 | 折線圖 | 線最好不超過 3～4 條 |
| 占整體比例 | 圓餅／堆疊長條 | 類別超過 5 個就別用圓餅 |
| 兩變數關係 | 散布圖 | 標出你要強調的那一群點 |

### 讓圖表更有力的三招
- **直接標結論**：標題別寫「營收圖」，寫「第三季營收成長 28%」。讓觀眾看標題就懂重點。
- **弱化其他、突出重點**：把要強調的那根長條上色，其餘全部變灰，視線自然被導過去。
- **刪掉干擾**：多餘的格線、外框、刻度、背景色都能拿掉——留下的越少，結論越清楚。

### 一張好圖表不需要解說
> [!QUOTE] 好的圖表不需要解釋，觀眾自己就看懂了。

如果你得花很多話去解釋一張圖，通常代表這張圖選錯型、或塞了太多資訊。回到第一步：這張圖只想讓觀眾看出哪一件事？

## 可執行要點
- [ ] 把簡報裡一張圖的標題，改成「這張圖的結論」那句話
- [ ] 挑一張長條圖，只把重點那根上色、其餘變灰
- [ ] 刪掉一張圖上多餘的格線、外框或刻度
- [ ] 檢查每張圖：類別是否太多、線是否太雜
